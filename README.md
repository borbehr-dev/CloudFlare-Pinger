# ⚡️ CloudFlare-Pinger

[![Deploy to Cloudflare](https://deploy.workers.cloudflare.com/button)](https://deploy.workers.cloudflare.com/?url=https://github.com/borbehr-dev/CloudFlare-Pinger)

Легковесный, адаптивный и полностью бесплатный сервис мониторинга доступности сайтов (Uptime Robot), работающий на технологиях **Cloudflare Workers** и **Hono**. 

Проект не требует серверов, баз данных и разворачивается за 1 минуту прямо в бессерверное облако Cloudflare Edge.

## ✨ Особенности

* **Serverless архитектура:** Работает полностью бесплатно на Cloudflare Workers.
* **Умный пинг (HEAD-запросы):** Проверяет ресурсы без скачивания лишнего трафика.
* **Защита от ложных падений:** Корректно обрабатывает защитные экраны капчи (Cloudflare, Google), не закрашивая живые сайты в красный цвет при получении статус-кодов `403` или `429`.
* **Cron-Trigger:** Автоматическое сканирование сайтов каждые 5 минут в фоновом режиме серверов Cloudflare.
* **Красивый интерфейс:** Современный SPA-дизайн с автоматической поддержкой **тёмной** и **светлой** темы в зависимости от настроек системы пользователя.
* **Ручной запуск:** Возможность мгновенно запустить проверку всех ресурсов прямо из панели кнопкой "Проверить сейчас".
* **Нативная JSON-конфигурация:** Никаких сложных YAML-парсеров, сборка работает "из коробки" на чистом JSON.

## 🚀 Быстрый старт

### 1. Подготовка репозитория
Клонируйте репозиторий и установите зависимости:
```bash
git clone https://github.com/borbehr-dev/CloudFlare-Pinger.git
cd CloudFlare-Pinger
npm install
```

### 2. Настройка списков ресурсов

* **`config.json`**: Основные настройки приложения (название, таймауты, доступы).
* **`list.json`**: Список сайтов, которые необходимо мониторить. Просто добавьте свои объекты:
```json
[
  {
    "id": 1,
    "name": "Google",
    "url": "https://www.google.com"
  },
  {
    "id": 2,
    "name": "Яндекс",
    "url": "https://ya.ru"
  }
]

```



### 3. Локальный запуск (Разработка)

Для тестирования скрипта на локальной машине выполните:

```bash
npm run dev

```

Интерфейс будет доступен по адресу `http://127.0.0.1:8787`.

### 4. Деплой в облако Cloudflare

Авторизуйте CLI-утилиту Wrangler и отправьте проект в продакшн:

```bash
npx wrangler login
npm run deploy
```

## 🛠 Технологический стек

* **Runtime:** Cloudflare Workers (V8 JavaScript Engine)
* **Framework:** Hono (Ультра-быстрый веб-фреймворк для Edge)
* **Bundler:** ESBuild (встроен в Wrangler)
* **Frontend:** Чистый HTML5 / CSS3 (Ванильный JS, CSS @media `prefers-color-scheme`, FontAwesome иконки)
